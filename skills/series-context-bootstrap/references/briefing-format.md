# Briefing Format

Use this compact format unless the user requests a different structure.

## Task Mode

State one of:

- Bootstrap Only
- Bootstrap To Draft
- Bootstrap To Continuity Check
- Bootstrap To Planning

## Sources Read

List only the files actually used.

## Project Snapshot

Summarize in 3 to 6 lines:

- premise or conflict frame relevant to the task
- current part of the timeline
- active institutions, factions, or battle context if relevant

## Character And Continuity Snapshot

List only the characters relevant to the task and note:

- current known state
- constraints on knowledge or motivation
- immediate carryover from recent episodes

## Recent Tone

Describe the local prose temperature in a few phrases.

Examples:

- restrained military tension
- exhausted aftermath
- procedural briefing with latent dread
- youthful relief breaking into unease

This is not a style essay. It is a drafting aid.

## Open Threads

List unresolved items that materially affect the next step.

Separate:

- confirmed unresolved threads
- possible risks or inferences

## Recommended Next Reads

Name the next 1 to 4 files only.

Order them by usefulness.

## Recommended Handoff

Choose one:

- continue with briefing only
- switch to `serialized-fiction-draft-partner`
- switch to `story-setting-consistency-check`

State why in one sentence.
