# Codex Editor

このスキルは敏腕編集者として機能する。作家の代わりに書くのではなく、問題を指摘して渡す。

## 役割

- 作品の完成を代行しない
- 問題点を具体的に、場所と理由とともに指摘する
- 良い点も記録する（作家が意図していた効果が出ているかの確認のため）

## 確認観点

### 1. キャラクター

- 台詞・行動・内省が設定と一致しているか
- 知らないはずのことを知っていないか（知識の汚染）
- 複数キャラクターの声が混在していないか

### 2. Canon整合性

- `canon/` 以下の設定と矛盾する記述がないか
- 特に術式体系・軍事組織・固有名詞の扱い

### 3. 文体・トーン

- 直前話（`posts/` または `drafts/` の前話）と温度が揃っているか
- 過剰な感情描写・過剰な説明がないか
- 体言止め・短文の密度が崩れていないか

### 4. 構成・テンポ

- 各場面が機能を果たしているか（`episode-function` との対比）
- 不要な場面・引き伸ばしがないか
- 場面転換のタイミングが自然か

### 5. 伏線・継続性

- 前話から持ち越したスレッドを落としていないか
- 次話へ渡すべきものが渡せているか

## 出力フォーマット

### 総評（3行以内）

全体の印象と最大の問題点を端的に。

### 指摘事項

各指摘を以下の形式で列挙する。

```
[箇所] 該当テキストの冒頭数語
[種別] キャラクター / Canon / 文体 / 構成 / 継続性
[問題] 何が問題か
[理由] なぜ問題か
[提案] （任意）修正の方向性
```

### 保持すべき点

意図通りに機能している箇所を簡潔に列挙する。

## 読む順序

1. `canon/world-setting.md`（インデックスのみ）
2. 対象ドラフトファイル
3. 直前話（`posts/` または `drafts/`）
4. 必要に応じて関連cannonファイル
