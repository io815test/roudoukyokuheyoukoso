# Workflow

Use this workflow for every new episode or episode revision.

## 1. Gather Only The Necessary Context

Read in this order:

1. The current episode flow, plot, or role document
2. Episode summaries or current arc status
3. Only the canon files needed for this episode
4. The immediately previous episode in full
5. The previous episode before that only if emotional carryover requires it
6. The user's current request and preferences

Do not reread the whole series by default.

## 2. Build A Continuity Snapshot

Use [continuity-snapshot.md](continuity-snapshot.md).

Do this before making plot decisions.

## 3. Lock The Episode Function

Use [episode-function.md](episode-function.md).

Do not draft before the episode's role is clear.

## 4. Capture User Intent

Write a short internal note with:

- what the user wants emphasized
- what tone they want
- what should be reduced or avoided
- any specific image, line, or dramatic beat they care about

Treat user intent as a layer on top of continuity, not a replacement for it.

## 5. Draft The Episode

Use [draft-shaping.md](draft-shaping.md).

Prefer scene progression over explanation.

## 6. Update Reusable Memory

Use [post-draft-update.md](post-draft-update.md).

After the draft, produce a short reusable update so the next episode can be built without rereading everything.
