# Serialized Fiction Draft Partner

Use this skill to produce new episode drafts without re-reading the whole project every time.

Standardize the workflow, not the story content.

Keep the story's continuity and temperature stable, then let the user's intent shape the episode, then draft prose that still leaves room for authorial enrichment.

## Workflow

1. Read only the minimum context needed in the order described in [references/workflow.md](references/workflow.md).
2. Build a compact continuity snapshot using [references/continuity-snapshot.md](references/continuity-snapshot.md).
3. Lock the episode's function using [references/episode-function.md](references/episode-function.md).
4. Match the local prose temperature using [references/tone-and-voice.md](references/tone-and-voice.md).
5. Draft the episode using [references/draft-shaping.md](references/draft-shaping.md).
6. After drafting, write a compact reusable update using [references/post-draft-update.md](references/post-draft-update.md).

## Reading Order

Read context in this order unless the user overrides it:

1. Current arc flow, episode role doc, or plot file
2. Running episode summaries or arc status
3. Only the canon files relevant to this episode
4. The immediately previous episode in full
5. The previous episode before that only if emotional carryover requires it
6. The user's current request and desired emphasis

Do not bulk-read the whole series unless blocked.

## Continuity Discipline

Preserve:

- timeline
- character knowledge
- emotional carryover
- canon constraints
- local tone
- unresolved threads

If the user asks for a continuity check, or if you suspect canon drift or unsupported assumptions, apply the workflow from [story-setting-consistency-check](../story-setting-consistency-check/SKILL.md) instead of improvising a looser review.

## Author Enrichment Principle

Do not overfinish every draft.

When useful, leave deliberate room for the author's own additions in:

- one or two key lines of dialogue
- one emotionally loaded image
- one transition paragraph
- the final cadence

If the user wants a fuller draft, write more completely. If the user wants a base draft to revise, bias toward clarity and continuity over polish.

## Context Economy

Prefer summaries over full-text rereads.

Expect or recommend these project-side memory files when available:

- `story-bible.md`
- `episode-summary.md`
- `current-arc-status.md`
- `character-state-tracker.md`

If they do not exist, recommend creating lightweight versions before the story grows too large for stable working context.
